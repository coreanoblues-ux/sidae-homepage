🎯 목표
배포 환경에서만 터지는 5대 원인(ENV, 쿠키, CORS, 라우팅, 혼합콘텐츠)을 한 번에 고친다.

## 1) 환경변수(DEV ↔ PROD) 정합
- .env.production (또는 배포 환경 변수) 설정:
  FRONT_ORIGIN=https://sidae-edu.com
  API_ORIGIN=https://sidae-edu.com
  COOKIE_DOMAIN=.sidae-edu.com
- 프론트 API 베이스는 “상대경로”로 통일해 배포/프리뷰 모두 OK:
  const API = import.meta.env.VITE_API_BASE_URL ?? '/api';

## 2) 쿠키(배포 도메인에서만 빠지는 문제 방지)
- 로그인/로그아웃 쿠키 옵션을 배포용으로 고정:
  const cookieOpts = {
    httpOnly: true,
    secure: true,
    sameSite: 'None',
    domain: process.env.COOKIE_DOMAIN, // .sidae-edu.com
    path: '/'
  };
  res.cookie('sid', token, { ...cookieOpts, maxAge: 1000*60*60*24*7 });
  // logout
  res.clearCookie('sid', cookieOpts); res.cookie('sid','', { ...cookieOpts, maxAge: 0 });

## 3) CORS/프록시 차이 제거
- 프리뷰는 Vite dev proxy가 살려줘서 되지만, 배포 후에는 CORS로 막힘 → 서버에 명시:
  app.set('trust proxy', 1);
  app.use(cors({ origin: process.env.FRONT_ORIGIN, credentials: true }));
- 프론트 fetch/axios는 무조건 credentials:'include'

## 4) SPA 라우팅 순서 고정(배포에서 HTML 내려오던 문제 방지)
  // 반드시 이 순서
  app.use('/api', apiRouter);                 // ① API 먼저
  app.use(express.static('dist'));            // ② 정적
  app.get('*', (req,res,next)=>{              // ③ SPA fallback
    if (req.path.startsWith('/api')) return next();
    res.sendFile(path.join(__dirname,'dist/index.html'));
  });

## 5) 혼합콘텐츠(NAS http) 배포 차단 대응
- 배포 URL은 https → http NAS는 막힘. 프록시로 통일:
  <video src={`/proxy/video?u=${encodeURIComponent(nasUrl)}`} controls />
- 서버 프록시(/proxy/video)는 Range 헤더 전달 + CORS 허용 유지.

## 6) 경로 보존 리다이렉트(도메인 강제 시 루프 방지)
  app.use((req,res,next)=>{
    const host = req.headers.host || '';
    const target = 'sidae-edu.com';
    const https  = req.headers['x-forwarded-proto'] === 'https';
    if (!https || host !== target) return res.redirect(301, `https://${target}${req.url}`);
    next();
  });

## 7) 배포 전 자가 점검(필수 3가지)
- curl -I https://sidae-edu.com/api/auth/me               → 200/401 JSON (HTML 금지)
- curl -I https://sidae-edu.com/_superadmin                → 200 text/html (index.html)
- curl -I "https://sidae-edu.com/proxy/video?u=http://…"   → 200/206 + Content-Type: video/mp4

## 8) 흔한 하드코딩 제거
- 소스에서 아래 문자열 검색해 모두 제거/치환:
  rg -n "http://localhost|127.0.0.1|replit\.app|riker\.replit\.dev" src server
  // 있으면 API = '/api'로, 외부 절대경로는 FRONT_ORIGIN/ENV 사용
