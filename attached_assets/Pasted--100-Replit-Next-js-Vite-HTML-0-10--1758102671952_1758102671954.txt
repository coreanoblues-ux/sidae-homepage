0) 10초 점검표 (증상→원인 힌트)

Network 탭 404 → 경로/폴더 위치 오류

403/401 → NAS 권한/토큰/핫링크 방지

CORS 에러 → 외부 도메인에서 차단

Mixed Content(http 리소스) → HTTPS 페이지에서 http 이미지 차단

200인데 MIME이 text/html → 빌드/라우팅이 이미지 대신 HTML을 돌려줌(경로 잘못됨)

/_next/image?url=… 400/404 → Next Image 최적화 설정/허용 도메인 문제

1) 가장 흔한 실수 5가지 (바로 고치기)
(A) 프로젝트 안의 이미지라면

/public 폴더에 이미지를 넣는다.

코드에서는 절대경로로:

// Next.js / Vite 공통
<img src="/images/jws-hero.jpg" alt="원장 정우석" />


❌ /public/images/... 라고 쓰면 안 보임. public은 경로에 쓰지 않음.

Next.js에서 next/image를 쓴다면:

import Image from "next/image";
<Image src="/images/jws-hero.jpg" alt="원장 정우석" width={1200} height={800} priority />

(B) 외부/NAS 링크를 쓰는 경우

먼저 <img>로 직빵 테스트

<img src="https://your-nas.example.com/path/photo.jpg" alt="테스트" />


이게 보이면 경로/권한은 정상. 이 상태에서만 next/image로 바꿔.

next/image 최적화 설정(외부 도메인 허용): next.config.js

/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    // 둘 중 하나(또는 둘 다) 사용
    domains: ["your-nas.example.com"], // 간단
    remotePatterns: [
      { protocol: "https", hostname: "**.your-nas.example.com" }, // 서브도메인 포함
    ],
    // Replit에서 static export 또는 프록시 곤란하면 임시로:
    // unoptimized: true,
  },
};
module.exports = nextConfig;


CORS/핫링크 방지 확인
NAS가 Access-Control-Allow-Origin을 안 주거나, Referer 차단하면 로드 실패.
가능하면 NAS에서 다음 중 하나:

CORS 허용(최소한 사이트 도메인 허용)

시간제 토큰 URL 사용

서버 프록시로 우회(아래 예시)

(C) HTTPS 혼합 콘텐츠

사이트가 https://인데 이미지가 http://면 차단됨.
→ NAS도 https URL을 쓰거나, 서버 프록시로 https화.

(D) 파일명/경로 문제

공백/한글/대소문자 혼용, #, ? 같은 문자 포함 시 인코딩 이슈 많음.
→ 영문/숫자/하이픈으로 파일명 단순화.

(E) Content Security Policy(CSP)

HTML <meta http-equiv="Content-Security-Policy" ...>가 엄격하면 외부 이미지 차단.

<!-- 최소한 이렇게: -->
<meta http-equiv="Content-Security-Policy" content="img-src 'self' data: https:;">


필요하면 도메인 추가: img-src 'self' https://your-nas.example.com data:;

2) 실패 원인 정확히 집는 방법 (DevTools 1분 루틴)

크롬 DevTools → Network 탭 → 문제가 되는 이미지의 요청 클릭

Status Code/URL 확인

404면 경로 오타/폴더 위치

403이면 NAS 권한/토큰/리퍼러

200인데 Response Headers의 content-type이 text/html → 라우팅이 HTML을 반환(경로 오류)

Console 탭에서 CORS/Mixed Content 메시지 체크

요청 URL을 새 탭에서 직접 열어보면, NAS가 로그인 페이지로 302 리다이렉트하는지도 확인

3) 바로 복붙 가능한 “확정 해결” 스니펫들
(1) Next.js 서버 프록시로 NAS 이미지 안전하게 불러오기

외부 이미지가 CORS/권한 문제를 낼 때, 서버에서 안전하게 가져다 주는 API 라우트를 하나 만듭니다. (App Router)

src/app/api/img/route.ts

export async function GET(req: Request) {
  const u = new URL(req.url).searchParams.get("u");
  if (!u) return new Response("missing u", { status: 400 });

  const upstream = await fetch(u, {
    // 필요 시 헤더 추가(토큰, Referer 등)
    // headers: { Authorization: `Bearer ${process.env.NAS_TOKEN}` }
  });

  if (!upstream.ok) {
    return new Response(`upstream ${upstream.status}`, { status: 502 });
  }

  const contentType = upstream.headers.get("content-type") ?? "image/jpeg";
  return new Response(upstream.body, {
    headers: {
      "Content-Type": contentType,
      "Cache-Control": "public, max-age=3600",
    },
  });
}
